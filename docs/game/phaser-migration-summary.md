# Phaser Overworld Migration — Summary & Test Record

**Branch:** `feat/phaser-overworld` (base `55803b8`)
**Date:** 2026-06-24
**What it does:** Replaces the inline-SVG island overworld with a Phaser 4 WebGL game (Tiled tilemap, Arcade physics, sprite animation, scene system) and ports the hop-and-collect minigame to a real Phaser platformer — while keeping React 19 for all HUD, menus, dialogue, and the other 8 minigames.

Plan: [docs/superpowers/plans/2026-06-24-phaser-rpg-migration.md](../superpowers/plans/2026-06-24-phaser-rpg-migration.md)
Engine capabilities / what to add next: [docs/game/engine-capabilities.md](./engine-capabilities.md)

---

## What shipped (by area)

| Area | Result |
|---|---|
| **Engine** | Phaser 4.2 mounted in React via `<PhaserGame>` (StrictMode-safe mount/destroy); typed `EventBus` singleton bridges React ↔ Phaser; `BootScene`/`OverworldScene`/`HopScene`. |
| **World** | Tiled tilemap (`overworld.tmj`, generated by `scripts/gen-overworld-map.mjs`): elliptical island, beach, radial paths, ocean collision. Runtime-generated tileset + player spritesheet (real Tiled/Aseprite art drops in later). |
| **Look** | Static full-island camera (matches old SVG framing). Themed landmark structures drawn in Phaser (museum, tram, lantern pavilion, fountain, rope bridge, flowers, lagoon, dock+boat, lighthouse). DOM Lucide medallion pins (labels, lock/check, objective ring, Glint) overlaid. Juice: 1.6× avatar, swaying palms, drifting clouds, objective sparkle — all motion-gated. |
| **State** | Transient `world` slice added to the Zustand store, excluded from persistence; the RAF/loop no longer drives React `setState` every frame. |
| **Audio** | `useGameAudio` hook (Howler) with SFX sprites; placeholder music loop disabled (`MUSIC_ENABLED=false`) until a real track exists. |
| **Minigame** | `treasureHop` (the chapter-4 platformer) ported to `HopScene` with Arcade physics (gravity, jump-when-grounded, platform collision, collectibles, goal-win). React `HopGame` wrapper keeps the Jump/Move-left/Move-right DOM controls and emits `hop:input`; scene emits `hop:done`. Other 8 minigames unchanged. |
| **Old SVG world** | `IslandMap.tsx` removed (preserved in git at `1c9db04`). |

## Architecture

```
React 19 + Vite + Wouter + TanStack Query          ← unchanged shell
├─ Zustand gameStore (+ transient `world` slice)    ← save/meta + world state (source of truth)
├─ EventBus (typed singleton)                        ← React ↔ Phaser bridge
├─ OverworldScene (Phaser, static-fit camera)        ← island art + avatar + landmark zones
│    └─ WorldHud (React DOM overlay)                 ← medallion pins, D-pad, Explore prompt
└─ HopScene (Phaser, Arcade physics)                 ← platformer minigame
     └─ HopGame (React wrapper)                       ← Jump/Move controls + win wiring
```

## Final review

A whole-branch review (Opus) found **no Critical issues** — the riskiest surface (Phaser/React lifecycle: scene teardown, EventBus + RESIZE listener cleanup, StrictMode double-mount safety, persistence exclusion) is implemented correctly, and all project constraints (no `any`, Phaser isolated to `src/game/**`, asset URLs via `BASE_URL`, aria-labels, motion gating) pass. The dead-wiring findings were fixed (commit `841bf7e`): SFX is now actually emitted (arrive/select), and the unused `world:moved` event + `world` store slice were removed. Final: unit 25/25, e2e 14/14, build OK.
